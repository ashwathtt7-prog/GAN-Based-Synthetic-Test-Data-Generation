LLM Loop 2 — Reasoning via graph traversal: The LLM never holds the full schema in memory. It calls Neo4j tools on-demand: get_table_schema(), get_relationships(), get_downstream_tables(), get_abbreviation(). It self-validates its output against the graph, retries up to 3 times, then routes to human review. PostgreSQL caches all decisions — columns already classified are never re-sent to the LLM.

LLM Loop 4 — Failure diagnosis: When a check fails the LLM doesn't just say "regenerate." It reads the failure report, diagnoses the root cause, writes a corrected strategy to PostgreSQL, and triggers targeted re-generation of only the affected domain.
